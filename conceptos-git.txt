Conceptos básicos de Git
========================

Repositorio:
Un repositorio es el espacio donde se almacenan todos los archivos de un proyecto junto
con el historial completo de cambios que se han realizado. Puede estar guardado de forma
local en el computador o de forma remota en plataformas como GitHub, lo que permite
acceder al proyecto desde cualquier lugar y compartirlo con otras personas.

Commit:
Un commit es un registro o fotografía del estado de los archivos en un momento específico.
Cada vez que se hace un commit, Git guarda qué archivos cambiaron, qué se modificó en
ellos y un mensaje descriptivo que explica el motivo del cambio. Esto permite llevar un
historial detallado de la evolución del proyecto y volver a versiones anteriores si es necesario.

Push:
El push es la acción de enviar los commits realizados localmente (en el computador o en
Codespaces) hacia el repositorio remoto en GitHub. Es el paso que hace que los cambios
queden visibles para otras personas o que queden respaldados en la nube. Sin hacer push,
los cambios solo existen en el entorno local y no se reflejan en GitHub.
